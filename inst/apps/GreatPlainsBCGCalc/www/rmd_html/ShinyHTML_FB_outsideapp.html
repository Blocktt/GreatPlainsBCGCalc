Prepare an input file outside of the app

If you choose to create the input file outside of the app (versus using the ‘File Builder’), the comma-separated file (CSV) needs to have all the required fields listed in the tables below depending on the biological community. Column names must match the spelling, symbology and capitalization scheme exactly as shown.

There is one more required field (EXCLUDE) that is not shown in the tables below. That column will be added automatically when you run the BCG Calculator. The EXCLUDE field marks redundant (non-distinct) taxa, using the same criteria that were used for BCG development. Redundant taxa are excluded from richness metric calculations but are included in the composition (percent individuals) metrics.

IMPORTANT: Make sure you are using the proper Operational Taxonomic Unit (OTU) and taxa traits/attributes for your metric calculations. The Taxa Translator and Attribute tables (and associated metadata) that were used for BCG development can be downloaded (Taxa Translator Files) (look for “GreatPlains_BCG_*” files). Note, that traits are not required for each record (i.e., trait fields allow null values).

Input file requirements for bugs

Input file requirements for bug BCG model.
Column Name  Data Type Domain (allowable entries) Null Allowed Comment
SAMPLEID text NA No Unique sample identifier (typically comprised of the site name, sample date, and replicate number).
TAXAID text NA No Taxonomic identification of organisms in each sample.
N_TAXA numeric non-negative real number No Number of individuals (not number of taxa).
NONTARGET text TRUE, FALSE No Non-target taxa should be marked “TRUE”.
FFG text CG, CF, PR, SC, SH, PH;PR Yes Primary Functional Feeding Group: Collector-Gatherer (CG), Collector-Filterer (CF), Piercer-Predator (PH;PR), Predator (PR), Scraper (SC), or Shredder (SH). Multiple entries are allowed and should be separated with a SEMICOLON rather than a comma or space.
HABIT text BU, CB, CN, SK, SP, SW Yes Habit types: Burrowers (BU), Climbers (CB), Clingers (CN), Skaters (SK), Sprawlers (SP), and Swimmers (SW). Multiple entries are allowed and should be separated with a SEMICOLON rather than a comma or space.
BCG_ATTR text 1, 2, 3, 4, 5, 6 Yes BCG Attribute assigned to each taxon. A single taxon can only have one entry.
TOLVAL numeric number ranging from 0-10 Yes Taxa with tolerance values ranging from 0 to 3 are considered ‘intolerant’ to anthropogenic disturbance and are included in the ‘intolerant’ metric calculations. Those with values ranging from 7 to 10 are included in the ‘tolerant’ metrics.
PHYLUM text NA Yes Entries should be consistent with the phylogeny.
ORDER text NA Yes Entries should be consistent with the phylogeny.
FAMILY text NA Yes Entries should be consistent with the phylogeny.

Input file requirements for fish

Input file requirements for fish BCG model.
Column Name  Data Type Domain (allowable entries) Null Allowed Comment
INDEX_CLASS text Small_47c, Large_47c, Small_Not_47c, Large_Not_47c No Index class determined by Level IV Ecoregion and watershed area (Sq. km): Small_47c = Ecoregion 47c and <100sqkm; Large_47c = Ecoregion 47c and >= 100sqkm; Small_Not_47c = All ecoregions not 47c and <100sqkm; Large_Not_47c = ecoregions not 47c and >= 100sqkm.
SAMPLEID text NA No Unique sample identifier (typically comprised of the site name, sample date, and replicate number).
TAXAID text NA No Taxonomic identification of organisms in each sample. Note, use scientific name here (e.g., Lepomis macrochirus) rather than common name.
N_TAXA numeric NA No Number of individuals.
FAMILY text NA Yes Entries should be consistent with the phylogeny.
GENUS text NA Yes Entries should be consistent with the phylogeny.
BCG_ATTR numeric 2, 3, 4, 5 Yes Biological Condition Gradient (BCG) taxa attribute assignments for the Great Plains BCG. 2 = highly sensitive; 3 = sensitive ubiquitous; 4 = moderately tolerant; 5 = highly tolerant.
BCG_ATTR2 text 2_MIDDLE, 3_BETTER, 3_MIDDLE, 3_WORSE, 4_BETTER, 4_WORSE, 5_MIDDLE. Yes BCG attribute assignments that accommodate sub-levels.
BCG_NUM numeric 2.0, 3.0, 3.5, 4.0, 4.5, 5.0 Yes Continuous numerical equivalent of BCG_ATTR2 field. 2_MIDDLE = 2.0; 3_BETTER = 3.0; 3_MIDDLE = 3.0; 3_WORSE = 3.5; 4_BETTER = 4; 4_WORSE = 4.5; 5_MIDDLE = 5.0.
NATIVE text NATIVE, NON_NATIVE Yes Specifies whether a taxon is native or non-native to the Great Plains region as a whole. Enter "NATIVE" for native taxa, and "NON_NATIVE" for non-native taxa.
TROPHIC text HB, IV, OM, TC Yes Trophic type: Herbivore (HB), Invertivore (IV), Omnivore (OM), top-carnivore (TC). A single taxon can only have one entry.
HABITAT text B, F, W (separate entries with ";") Yes Habitat type: benthic (B), fluvial (F), and water-column (W). A single taxon can have multiple entries.
REPRODUCTION text BEARER, BROADCASTER, COMPLEX NEST, LITHOPHIL, MIGRATORY, SIMPLE NEST (separate entries with ";") Yes Reproduction type. Only LITHOPHIL is used for metric calculations. A single taxon can have multiple entries.
